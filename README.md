# ライブキャプション文字起こし

Windows の純正機能「ライブ キャプション」に表示された文字を読み取り、メモ帳とテキストファイルに保存するためのツールです。

リアルタイムで無理に入力し続けるのではなく、ライブ キャプションはそのまま画面で見て、メモ帳をクリックしたタイミングで文字起こし内容を更新します。

## できること

- Windows 純正のライブ キャプションを起動する
- 文字起こし結果を `transcripts` フォルダへ保存する
- メモ帳を同時に開く
- メモ帳を前面にしたタイミングで、現在までの文字起こしを反映する
- 停止用ショートカットから終了し、最新の文字起こしファイルを開く

## 必要なもの

- Windows 11
- Windows のライブ キャプション機能
- 日本語ライブ キャプション
- PowerShell
- メモ帳

## 初回セットアップ

GitHub からダウンロードした直後は、ショートカットがまだありません。

フォルダを開いて、PowerShell で次を実行してください。

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\.app\Install-Shortcuts.ps1
```

成功すると、次の2つのショートカットが作成されます。

- `ライブキャプション文字起こし`
- `ライブキャプション文字起こし停止`

## 使い方

1. `ライブキャプション文字起こし` を起動する
2. ライブ キャプションとメモ帳が開く
3. 音声を流す
4. メモ帳をクリックすると、そこまでの文字起こしが更新される
5. 終わるときは `ライブキャプション文字起こし停止` を起動する

停止すると、最新の文字起こしファイルがメモ帳で開きます。

## 出力先

文字起こしファイルは `transcripts` フォルダに保存されます。

ファイル名の例:

```text
caption-20260609-170000.txt
```

## 操作のコツ

- リアルタイム表示はライブ キャプション側を見る
- メモ帳は「確認したいとき」にクリックする
- 別作業中はメモ帳を前面にしなければ、勝手に入力されにくい
- 終了するときは、メモ帳を閉じるだけでなく停止ショートカットを使う

## 制限

このツールは Microsoft 公式のエクスポート API を使っているわけではありません。

Windows の画面上に出ているライブ キャプションの文字を読み取っているため、次のような制限があります。

- ライブ キャプションに表示される前の音声は取れない
- ライブ キャプション側で文字が消えた後に読み取れなかった分は復元できない
- 長時間の連続使用では、途中で保存状況を確認するのがおすすめ
- Windows UI を使うため、クラウド環境では実機テストできない

## トラブルシューティング

### メモ帳が更新されない

メモ帳を一度別ウィンドウの後ろにしてから、もう一度クリックしてください。

このツールは、メモ帳が前面になったタイミングで更新します。

### ライブ キャプションが開かない

手動で `Win + Ctrl + L` を押してライブ キャプションを起動してください。

### 文字が重複する

ライブ キャプションは途中の文章を後から修正することがあります。

このツール側でも重複を減らす処理を入れていますが、完全ではありません。気になる場合は、停止後に `transcripts` のテキストを確認してください。

### 途中で止めたい

`ライブキャプション文字起こし停止` を起動してください。

停止用ショートカットは、動いている文字起こし処理を止めて、最新ファイルをメモ帳で開きます。

## GitHub から使う人へ

`.lnk` ショートカットや `transcripts` の中身は GitHub には含めていません。

ダウンロード後は、必ず初回セットアップを実行してください。
