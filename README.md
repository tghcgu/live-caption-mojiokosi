# ライブキャプション文字起こし

Windows 11 の純正機能「ライブ キャプション」に表示された文字を読み取り、メモ帳とテキストファイルに保存するツールです。

ライブ キャプションは画面でリアルタイムに確認し、メモ帳をクリックしたタイミングで文字起こし内容を更新します。

## 先に大事なこと

GitHub からダウンロードしただけでは、まだ起動用ショートカットは使える状態になっていません。

最初に一度だけ、同梱のセットアップを実行してください。

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\.app\Install-Shortcuts.ps1
```

セットアップを実行すると、次の2つのショートカットが作られます。

```text
ライブキャプション文字起こし
ライブキャプション文字起こし停止
```

古いPCで作った `.lnk` ショートカットだけを新しいPCへコピーしても使えません。ショートカットにはPCごとのフォルダ場所が入っているため、新しいPCでは必ずセットアップをやり直してください。

## できること

- Windows のライブ キャプションを起動する
- メモ帳を同時に開く
- メモ帳をクリックしたタイミングで文字起こしを更新する
- 文字起こし結果を `transcripts` フォルダへ保存する
- 停止用ショートカットで終了し、最新の文字起こしファイルを開く

## 必要なもの

- Windows 11
- Windows のライブ キャプション機能
- 日本語ライブ キャプション
- PowerShell
- メモ帳

このツールは Windows 専用です。Mac、Linux、Codex Cloud などでは実機動作しません。

## GitHub からダウンロードして使う手順

1. GitHub のリポジトリページを開く

   <https://github.com/tghcgu/live-caption-mojiokosi>

2. 緑色の `Code` ボタンを押す

3. `Download ZIP` を押す

4. ダウンロードした ZIP ファイルを右クリックして、`すべて展開` を押す

5. 展開されたフォルダを開く

6. フォルダを開いた状態で、エクスプローラーのアドレスバーに `powershell` と入力して Enter を押す

7. PowerShell で次のコマンドを実行する

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\.app\Install-Shortcuts.ps1
```

8. 作成された `ライブキャプション文字起こし` を起動する

## Git を使って入れる場合

Git が使える人は、次のコマンドでも入れられます。

```powershell
git clone https://github.com/tghcgu/live-caption-mojiokosi.git
cd live-caption-mojiokosi
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\.app\Install-Shortcuts.ps1
```

## 使い方

1. `ライブキャプション文字起こし` を起動する
2. ライブ キャプションとメモ帳が開く
3. 音声や動画を再生する
4. メモ帳をクリックすると、そこまでの文字起こしが更新される
5. 終わるときは `ライブキャプション文字起こし停止` を起動する

停止すると、最新の文字起こしファイルがメモ帳で開きます。

## ファイルが保存されるタイミング

`transcripts` のテキストファイルに書き込まれるのは、次のタイミングです。

- メモ帳をクリックして文字起こしが更新されたとき
- 停止ショートカットで停止したとき

メモ帳を触らずに放置していると、ファイルの中身が古いまま(最初は空のまま)に見えますが、故障ではありません。文字起こし自体は裏で続いています。

停止ショートカットを使えば、そこまでの文字起こし全体が必ずファイルに保存されます。長時間使うときは、ときどきメモ帳をクリックして途中経過を保存しておくと安心です。

## どれを起動すればいいか

普段使うのはこの2つだけです。

```text
ライブキャプション文字起こし
ライブキャプション文字起こし停止
```

`.app` フォルダの中の PowerShell ファイルは、ショートカットから呼び出される内部ファイルです。普段は直接開かなくて大丈夫です。

ただし、`.app` フォルダは消さないでください。消すとショートカットが動かなくなります。

## 新しいPCで使うとき

新しいPCでは、次の順番で使ってください。

1. GitHub からこのリポジトリをダウンロードする
2. ZIP を `すべて展開` する
3. 展開したフォルダで PowerShell を開く
4. セットアップコマンドを実行する
5. 作成された `ライブキャプション文字起こし` を起動する

古いPCのショートカットをコピーする必要はありません。むしろコピーすると、古いPCのパスを見に行って起動しないことがあります。

## フォルダを移動したとき

このツールのフォルダを別の場所へ移動した場合は、もう一度セットアップコマンドを実行してください。

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\.app\Install-Shortcuts.ps1
```

ショートカットのリンク先が、新しいフォルダ場所に更新されます。

OneDrive の同期設定を変えたときも注意してください。デスクトップが OneDrive 同期に切り替わる(または解除される)と、フォルダの実際の場所が `C:\Users\ユーザー名\Desktop` と `C:\Users\ユーザー名\OneDrive\Desktop` の間で変わり、ショートカットが古い場所を見たままになります。この場合もセットアップの再実行で直ります。

## 新しい版に更新するとき

1. GitHub から新しい ZIP をダウンロードして展開する
2. 過去の文字起こしを残したい場合は、古いフォルダの `transcripts` を新しいフォルダへコピーする
3. 新しいフォルダでセットアップコマンドを実行する

Git で入れた場合は、フォルダ内で `git pull` を実行するだけで更新できます。フォルダの場所が変わっていなければ、セットアップのやり直しは不要です。

## 出力先

文字起こしファイルは `transcripts` フォルダに保存されます。

ファイル名の例:

```text
caption-20260609-170000.txt
```

`transcripts` の中身は GitHub には含めていません。別のPCへ過去の文字起こし結果も持っていきたい場合は、`transcripts` フォルダを別でコピーしてください。

## 操作のコツ

- リアルタイム表示はライブ キャプション側を見る
- メモ帳は確認したいタイミングでクリックする
- 別作業中はメモ帳を前面にしなければ、勝手に入力されにくい
- 終了するときは、メモ帳を閉じるだけでなく停止ショートカットを使う

## よくあるトラブル

### 起動しない

ショートカットのリンク先が古くなっている可能性があります。

展開したフォルダで PowerShell を開き、もう一度セットアップコマンドを実行してください。

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\.app\Install-Shortcuts.ps1
```

特に、OneDrive、デスクトップ、Downloads などへフォルダを移動した後は、セットアップのやり直しが必要です。

### セットアップコマンドでエラーになる

PowerShell が展開したフォルダで開かれているか確認してください。

正しい場所で開けている場合、PowerShell の表示はだいたい次のようになります。

```text
PS C:\Users\ユーザー名\Downloads\live-caption-mojiokosi-main>
```

`.app` が見つからないというエラーが出る場合は、別のフォルダで PowerShell を開いています。

### ライブ キャプションが開かない

手動で `Win + Ctrl + L` を押してライブ キャプションを起動してください。

初回は Windows 側でライブ キャプションの準備や言語設定が必要になる場合があります。

### メモ帳が更新されない

メモ帳をクリックしてください。

それでも更新されない場合は、一度別のウィンドウをクリックしてから、もう一度メモ帳をクリックしてください。

### 文字が重複する・表示と少し違う

ライブ キャプションは、一度表示した文章を後から書き直すことがあります。このツールは書き直しを検知して新しい方に置き換えますが、タイミングによってはまれに重複が残ることがあります。

「うん」が「分」になるような文字の違いは、ライブ キャプション側の音声認識によるものです。ツールでは直せません。

### 止めたい

`ライブキャプション文字起こし停止` を起動してください。

停止用ショートカットは、動いている文字起こし処理を止めて、最新ファイルをメモ帳で開きます。

## GitHub に含めていないもの

次のものは GitHub には含めていません。

- 作成済みショートカット `.lnk`
- 文字起こし結果 `transcripts`
- 一時ファイル

そのため、ダウンロード後は必ず初回セットアップを実行してください。

## 仕組み

このツールは、Windows が公式に用意している画面読み取りの仕組み(UI Automation。スクリーンリーダーなどが使うものと同じ)を使って、ライブ キャプションのウィンドウに表示された文字を約0.2秒ごとに読み取っています。

読み取った文字はつなぎ合わせて、ライブ キャプション特有の「書き直し」による重複を取り除いてから保存します。

- 音声の録音はしていません
- 読み取った文字をインターネットへ送ることもありません
- すべて自分のPCの中だけで完結します

## 使うときのマナー

ツール自体は公式の仕組みだけで動いていますが、何を文字起こしするかには注意してください。

- 会議を文字起こしする場合は、参加者に一言伝えておくのが安全です
- 動画や配信を自分用に文字起こしして保存するのは一般に問題ありませんが、その内容をネットに公開・配布すると著作権の問題になります

## 文字起こしの特性

仕様として、次のような動きをします。

- 「はい」「ええ」のような短い相づちも記録されます
- まったく同じ言葉を間を置かずに連続して言うと、画面上で区別できないため1回にまとまります
- ライブ キャプションが後から文章を書き直した場合は、新しい方に置き換えて記録します
- 同音異義語などの認識間違いは、ライブ キャプション側の音声認識によるものです

## 制限

このツールは Microsoft 公式のエクスポート API を使っているわけではありません。

Windows の画面上に出ているライブ キャプションの文字を読み取っているため、次のような制限があります。

- ライブ キャプションに表示される前の音声は取れない
- ライブ キャプション側で文字が消えた後に読み取れなかった分は復元できない
- 長時間の連続使用では、途中で保存状況を確認するのがおすすめ
- Windows UI を使うため、クラウド環境では実機テストできない
